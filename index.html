Digital Publishing prototypes

A sprint by sprint list of what we've created and what we've learnt from our research.

Sprint 0 (Inception)

  • Search datasets with filtering (Prototype v1)
  • Usability testing themes (Google doc)

Sprint 1

9 Nov to 16 Nov

  • Customise a dataset: Armed forces (Prototype v1)
  • Usability testing themes (Google doc)

Things we learnt:

  • “Customise this dataset” performed better than "review topics"
  • Users need meta data earlier in the process
  • Not all users are familier with all file types

Sprint 2

16 Nov to 23 Nov

  • Customise a dataset: Armed forces (Prototype v2)

Things we learnt:

  • Wording and labels are really important
  • Users need supporting information for context
  • ZIP folders can cause problems

Sprint 3

23 Nov to 7 Dec

  • Customise a dataset: Armed forces (Prototype v3)
  • Customise a dataset: Consumer price index (CPI) (Prototype v1)
  • Usability testing themes (Google doc)
  • Analytics themes (Google doc)

Things we learnt:

  • Geography is hard
  • Nested lists are probably the most complex pattern to solve
  • Users want to "search first"

Sprint 4

7 Dec to 21 Dec

  • Generate files user journey (Mock-up v1)
  • Generate files user journey (Diagram)
  • Home page geography journey (Mock-up v2)
  • Customise a dataset: Armed forces (with Geography) (Mock-up v2)
  • Observation (include/don't include) page (Mock-up v1)
  • Usability testing themes (Google doc)
  • Analytics: popular location searches (Google Sheet)

Things we learnt:

  • Some users really like our old websites
  • Generating custom data downloads might take a long time
  • Analytics suggests people search by city
  • Recruiting users for research is hard

Sprint 5

5 Jan to 18 Jan

  • Dataset page (and journey) (Mock-up v1)
  • Dataset page (and journey) (Mock-up v2)
  • UX pattern library
  • Design critique (Google doc)
  • Dataset survey (Google slides)
  • Customise a dataset: Armed forces (Prototype v4)

Things we learnt:

  • We should respond to the user within one second
  • “Users find some spreadsheets difficult to navigate” – contact centre
  • Some users expect locations to be ordered alphabetically
  • Nobody knows what special aggregate means

Sprint 6

18 Jan to 1 Feb

  • COICOP Special Aggregate (Prototype)
  • SAPE Multiple Geographic Hierarchies (Prototype)
  • Dataset metadata (Prototype)
  • Faceted search (Mock-up)
  • Single page selector 1 (Mock-up)
  • Single page selector 2 (Mock-up)

Things we learnt:

  • Users find 14px font size too small
  • Users expect Cancel button to take them back to the start
  • “I like to know what [geographic] levels are available on the dataset page”
  • Users expect to be shown their previous selections first when updating a customisation
  • Research suggests a unified pattern for nested lists and geographic hierarchies will work

Sprint 7

1 Feb to 14 Feb

  • Sprint 07 geography selector (Prototype)

Things we learnt:

  • Users need better content and terminology
